# Swiggy Restaurant SQL Case Study

![Swiggy SQL Case Study](Swiggy-Case-Study.jpg)

## 📌 Project Overview
Analyzed Swiggy restaurant dataset using SQL to uncover insights on ratings, pricing, cuisine distribution, and restaurant trends.

## 📊 Dataset
- Swiggy restaurant dataset containing restaurant names, cities, ratings, cost per person, and cuisines

## 🛠 Tools Used
- SQL (SQL Server)

## 🔍 Key Analysis
- Count of highly rated restaurants (rating > 4.5)
- City with the highest number of restaurants
- Restaurants with "Pizza" in their name
- Most common cuisine
- Average rating per city
- Highest priced recommended item by restaurant
- Top 5 most expensive restaurants
- Top 5 expensive restaurants (excluding Indian cuisine)
- Restaurants costlier than average
- Duplicate restaurants across multiple cities
- Average cost per person by restaurant
- Cuisine-wise restaurant count
- City-wise restaurant distribution
- City with the highest average rating
- Most expensive menu item overall
- Veg vs Non-Veg item distribution
- Cuisine with the highest average cost per person

## 📈 Key Insights
- Identified cities with high restaurant concentration  
- Found most popular cuisines across different regions  
- Observed pricing variations among restaurants and cuisines  
- Detected duplicate restaurant entries across multiple cities  
- Highlighted top-rated and premium restaurants  

## 📂 Files Included
- swiggy_case_study.sql → SQL queries used for analysis

## 🚀 Conclusion
This project demonstrates strong SQL skills including data aggregation, filtering, grouping, and deriving meaningful business insights from raw data.
